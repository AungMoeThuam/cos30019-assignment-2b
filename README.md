# Traffic-Based Route Guidance System (TBRGS)

COS30019 - Introduction to Artificial Intelligence (Assignment 2B)

This repository contains the skeletal structure for the Traffic-Based Route Guidance System (TBRGS). The system predicts traffic flow using TensorFlow-based deep learning models (LSTM and GRU) and integrates those predictions with an A\* pathfinding algorithm to calculate the fastest routes in the city of Boroondara.

---

## Training & Execution Strategy

> [!NOTE]
> **Google Colab for Training:** Because deep learning models (LSTM and GRU) are computationally intensive and train significantly faster with hardware acceleration, **all model training is designed to take place on Google Colab (utilizing a GPU/TPU runtime)**.
>
> Once training is completed on Colab, you will download the saved weights/models into your local `models_saved/` directory. Your local machine is only used to run the lightweight A\* pathfinder CLI (`A2B.py`) using these pre-trained models.

## 2. Installation & Prerequisites

### Python Version Warning

> [!IMPORTANT]
> **TensorFlow Compatibility:** TensorFlow currently does not officially support **Python 3.14.0** or above.
> To run this project locally, you must use a Python version between **3.9 and 3.13** (Python 3.10 to 3.12 is highly recommended for stability).

### Local Setup

1. Clone the repository to your local machine.
2. Create a virtual environment using a compatible Python version:
   ```bash
   # Windows PowerShell
   py -3.12 -m venv .venv
   .venv\Scripts\activate
   ```
3. Install the dependencies:
   ```bash
   pip install -r requirements.txt
   ```

## 3. Workflow & Usage

### Step 1: Data Preprocessing & Training on Google Colab

Since deep learning models are computationally heavy, training should be done on Google Colab:

1. Open the [notebooks/Colab_Training_Template.ipynb](notebooks/Colab_Training_Template.ipynb) notebook in Google Colab.
2. Clone your repository inside Colab and upload the raw Excel data to the `data/raw/` directory.
3. Run the notebook to train the LSTM and GRU models.
4. Download the resulting model checkpoints (e.g., `lstm_model.h5`, `gru_model.h5`) from the `models_saved/` directory in Colab and place them in your local `models_saved/` directory.

### Step 2: Running the CLI

Run the pathfinder command from your terminal:

```bash
python A2B.py <origin_node> <destination_node> <time> <model_type>
```

#### Parameters:

- `origin_node`: The starting SCATS site ID (e.g., `2000`).
- `destination_node`: The ending SCATS site ID (e.g., `2825`).
- `time`: The 24-hour departure time formatted as HHMM (e.g., `1100` for 11:00 AM).
- `model_type`: The prediction model to use (`LSTM`, `GRU`, or `RANDOM`).

#### Example command:

```bash
python A2B.py 2000 2825 1100 LSTM
```
